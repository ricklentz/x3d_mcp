# X3D Ecosystem Research & Existing Tools

Research conducted 2026-03-04. Findings for leveraging community works before building.

## Key Finding: No X3D MCP Server Exists

All existing 3D MCP servers focus on Three.js/glTF, OpenSCAD/STL, or app-specific tools (Blender, Rhino, SketchUp). **This is a greenfield project.**

## Existing X3D Python Packages

### `x3d` (x3d.py / X3DPSAIL) -- THE Official Package
- **PyPI**: https://pypi.org/project/x3d/
- **Version**: 4.0.65.0 (February 6, 2026 -- actively maintained)
- **Author**: Don Brutzman (Web3D Consortium)
- **License**: BSD-3-Clause
- **Source**: https://www.web3d.org/x3d/stylesheets/python/python.html
- **How it works**: Entirely autogenerated from X3DUOM via `X3duomToX3dPythonPackage.xslt`
- **Capabilities**:
  - Full Python Scene Access Interface Library
  - Programmatic scene creation, manipulation, serialization
  - Strict type-checking (`isValidSFBool()`, `isValidSFVec3f()`, `assertValidMFRotation()`)
  - Generates X3D XML output
  - Supports X3D versions 3.0-3.3 and v4.0 draft
- **Limitation**: XML output only -- no JSON encoding

### `pythonSAI`
- **GitHub**: https://github.com/coderextreme/pythonSAI
- **Last commit**: February 21, 2026 (active)
- **What it does**: PyJNIus-based Python bindings to X3DJSAIL (Java)
- **Note**: Requires Java runtime

### Other Python X3D packages (dormant/niche)
- `x3dase` (molecular viz, dormant since 2020)
- `x3d-python-mod` (tools to modify x3d.py, minimal activity)
- `x3d-pathway` (scientific data to X3D, dormant since 2016)

## Existing X3D JavaScript/TypeScript Packages

### `x_ite` + `x_ite-node` -- The Clear Leader
- **GitHub**: https://github.com/create3000/x_ite
- **Author**: Holger Seelig (create3000)
- **Stars**: ~78, but 16,520+ commits
- **Maintained**: Actively (updated January 2026)
- **License**: GPL-3.0
- **Capabilities**:
  - Full X3D browser implementation in JS/WebGL
  - Parses X3D XML, JSON, Classic VRML, VRML97, glTF, OBJ, STL, PLY, SVG
  - Full SAI (Scene Access Interface): `createNode()`, `createScene()`
  - Serialization: `toXMLString()`, `toJSONString()`, `toVRMLString()`
- **`x_ite-node`**: Headless Node.js version (no browser/Electron needed)
  - This could be the foundation for a TypeScript MCP server

### `x3d-tidy` (npm)
- CLI tool by same author
- Converts between X3D encodings (XML, JSON, Classic VRML)
- Beautifies/minimizes X3D

### `x3d-image` (npm)
- CLI tool to render X3D scenes to PNG/JPEG

### `x3dvalidate` (npm)
- **GitHub**: https://github.com/coderextreme/x3dvalidate
- **Author**: John Carlson (Web3D contributor)
- Validates X3D JSON files against X3D JSON Schema 4.0
- Uses Ajv (JSON schema validation engine)
- `npx x3dvalidate file.x3dj`

### `X3DJSONLD`
- **GitHub**: https://github.com/coderextreme/X3DJSONLD
- X3D JSON loader, converts JSON to X3D XML/Python/Java

### X3DOM (npm: `x3dom`)
- **GitHub**: https://github.com/x3dom/x3dom (847 stars)
- **Status**: INACTIVE -- last release August 2023
- Uses standard DOM manipulation, no dedicated scene API

## Python Validation Libraries

### `lxml` (v6.0.2, BSD-3)
- https://pypi.org/project/lxml/
- **Best for**: XSD validation of X3D XML
- 100M+/month downloads, actively maintained
- `lxml.etree.XMLSchema` validates against X3D XSD
- Also supports DTD, RelaxNG, ISO-Schematron
- ~42x faster than xmlschema

### `xmlschema` (v4.3.1, MIT)
- https://pypi.org/project/xmlschema/
- Pure Python XSD validator (no C deps)
- Full XSD 1.0 and 1.1 support
- Can decode XML to Python dicts/JSON (useful for X3D XML-to-JSON)
- Slower than lxml but more standards-compliant

### `jsonschema` (v4.26.0, MIT)
- https://pypi.org/project/jsonschema/
- Validates X3D JSON against `x3d-3.3-JSONSchema.json` or `x3d-4.0-JSONSchema.autogenerated.json`

## MCP SDK & Architecture

### Official SDKs
- **Python**: `mcp` on PyPI (v1.7.1), `FastMCP` class built-in
- **TypeScript**: `@modelcontextprotocol/server` on npm, uses Zod v4

### Protocol
- Current spec: `2025-11-25`
- Transport: stdio (local) or Streamable HTTP (remote)
- JSON-RPC 2.0 based
- Three primitives: **Tools** (model-controlled), **Resources** (app-controlled), **Prompts** (user-controlled)

### FastMCP (Python) Pattern
```python
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("X3D MCP Server")

@mcp.tool()
def generate_scene(description: str, format: str = "xml") -> str:

```

### Design Pattern: Workflow-Based
From Klavis research: "Think of MCP tools as tailored toolkits that help an AI achieve a particular task, not as API mirrors." Higher-level tools are more reliable than many granular ones.

## X3DUOM: The Foundation

The `X3dUnifiedObjectModel-4.0.xml` file:
- Auto-generated, contains EVERY node, field, type, default, containerField
- Sections: `SimpleTypeEnumerations`, `AbstractNodeTypes`, `AbstractObjectTypes`, `ConcreteNodes`, `Statements`
- Each `ConcreteNode` has: component, fields (name, type, accessType, default, tooltip), inheritance
- URL: https://www.web3d.org/specifications/X3dUnifiedObjectModel-4.0.xml
- The official `x3d.py` package is generated FROM this file via XSLT

### X3D Validation Resources Available
| Resource | URL |
|----------|-----|
| XSD 4.0 | https://www.web3d.org/specifications/x3d-4.0.xsd |
| DTD 4.0 | https://www.web3d.org/specifications/x3d-4.0.dtd |
| XSD 3.3 | https://www.web3d.org/specifications/x3d-3.3.xsd |
| JSON Schema 3.3 | https://www.web3d.org/specifications/x3d-3.3-JSONSchema.json |
| JSON Schema 4.0 (auto) | https://www.web3d.org/specifications/x3d-4.0-JSONSchema.autogenerated.json |
| X3DUOM 4.0 | https://www.web3d.org/specifications/X3dUnifiedObjectModel-4.0.xml |

| Schematron XSLT | `X3dSchematronValidityChecks.xslt` (SourceForge) |

## Docker Validation Options

### `leplusorg/xml` Docker Image
- xmllint + Saxon HE in one image
- XSD validation: `docker run --rm -v "$(pwd):/tmp" leplusorg/xml xmllint --schema /tmp/x3d-4.0.xsd /tmp/scene.x3d --noout`
- XSLT (Schematron): `docker run --rm -v "$(pwd):/tmp" leplusorg/xml java -jar /opt/saxon/run.sh -s:/tmp/scene.x3d -xsl:/tmp/X3dSchematronValidityChecks.xslt`

### `atomgraph/saxon` Docker Image
- Dockerized Saxon-HE for XSLT 3.0 transforms
- For applying Schematron XSLT to X3D files

### `isaitb/xml-validator` (EU Interoperability Test Bed)
- Docker image with REST API for XSD + Schematron validation
- Configurable via `config.properties`
- Has both web UI and REST API
- Could be configured for X3D validation

### Python-native (no Docker needed)
- `lxml` handles XSD validation directly
- `lxml.isoschematron` handles Schematron validation
- `xmlschema` for pure-Python XSD validation
- `jsonschema` for JSON Schema validation

## Decision Matrix: Python vs TypeScript

| Factor | Python | TypeScript |
|--------|--------|-----------|
| Official MCP SDK | FastMCP (mature) | @modelcontextprotocol/server (mature) |
| X3D generation | `x3d.py` (official, auto-generated from X3DUOM) | `x_ite-node` (full SAI, GPL-3.0) |
| XML validation | `lxml` (fast, XSD+Schematron) | `xsd-schema-validator` (requires Java) |
| JSON validation | `jsonschema` (mature) | `x3dvalidate` (Ajv-based, X3D-specific) |
| X3D JSON output | Must build (x3d.py is XML-only) | `x_ite-node` has `toJSONString()` built-in |
| Conversion tools | Must build | `x3d-tidy` handles all encodings |
| License concern | BSD (all libs) | GPL-3.0 (x_ite) |

**Python advantage**: Simpler stack, official `x3d.py` package, `lxml` for validation, `FastMCP` is ergonomic.
**TypeScript advantage**: `x_ite-node` gives us XML+JSON+VRML output plus parsing for free, `x3d-tidy` for conversion.
**License concern**: `x_ite` is GPL-3.0. If we use it, our MCP server must also be GPL-3.0 or compatible.
